# Contributing to InnovateSwarm Instructions

Thank you for your interest in contributing to **InnovateSwarm Open Instruction Pages**!  
This repository contains high-quality, ready-to-use multi-agent prompt templates and instruction systems.

We welcome contributions that improve existing pages or add new powerful agentic workflows.

## Ways to Contribute

- Fixing typos, grammar, or formatting issues
- Improving clarity, structure, or instructions
- Adding better examples or use cases
- Suggesting new instruction pages (new agent systems)
- Enhancing "Best For" sections, tags, or Quick Start Prompts
- Improving the main README or documentation

## How to Contribute

### 1. Fork & Clone the Repository

```bash
git clone https://github.com/InnovateSwarm/InnovateSwarm-Instructions.git
cd InnovateSwarm-Instructions
````

### 2. Create a New Branch

```bash
git checkout -b improvement/swarm-tank-clarity
# or
git checkout -b new-instruction/my-awesome-system
````

### 3. Follow Our Style Guidelines

All instruction pages should maintain a consistent structure:

Clear, compelling title
Short description
Best For section (bullet points)
How to Use (detailed step-by-step protocol)
Quick Start Prompt (easy to copy-paste)
Tags (at the bottom)
Clean footer with credit to InnovateSwarm

### Please:

Use professional yet engaging language
Keep instructions precise and actionable
Use consistent markdown formatting
Test the prompt in an agent framework before submitting

### 4. Make Your Changes

Edit existing pages inside their folders
For new instruction pages, create a new folder with a README.md

### 5. Commit & Push

```bash
git add .
git commit -m "Improve clarity in Swarm Tank instructions"
git push origin improvement/swarm-tank-clarity
````

### 6. Open a Pull Request

Go to GitHub and open a Pull Request with:
A clear description of your changes
Why the change improves the repository
Any screenshots or example outputs (optional but helpful)

### Code of Conduct

Be respectful and constructive
Assume positive intent
Focus on making the instructions better for everyone

###Questions?
Feel free to open an issue if you have questions, want feedback on a new idea, or need help getting started.

We appreciate every contribution — big or small!
